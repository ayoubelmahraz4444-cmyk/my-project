Approche

Trois principes, aucune exception

Minimal

Pas de nœud inutile

Chaque étape d'un workflow doit justifier son existence. Un pipeline propre se débogue en minutes, pas en heures.

Résilient

Tout finit par échouer

Retries, upserts, validation des clés API en amont — je conçois pour l'échec transitoire, pas seulement pour le cas idéal.

Traçable

Visible à chaque étape

Slack en cas de succès, email en cas d'erreur, logs lisibles. Un automatisme silencieux est un automatisme qu'on ne peut pas réparer.

Ce que j'ai construit récemment

Dernier projet · en production

Automatisation de leads de bout en bout

Pipeline n8n qui capte un lead entrant, l'enrichit via un agent IA (catégorisation, score d'importance, langue détectée), génère un vecteur d'embedding, le stocke dans Supabase pour la recherche sémantique, puis notifie l'équipe commerciale en temps réel. Inclut une logique de retry automatique sur les erreurs transitoires de l'API et une déduplication par upsert.

  • n8n
  • Gemini API
  • Supabase
  • Slack
  • Gmail

Pipeline RAG multi-format

Ingestion automatique de PDF, Excel, CSV, images et fichiers Word depuis Google Drive, avec OCR de secours pour les documents scannés et indexation vectorielle dans Supabase.

  • n8n
  • OCR
  • Vector DB

Agent conversationnel documentaire

Chatbot RAG qui répond uniquement à partir des documents internes d'une entreprise, avec mémoire de conversation et refus explicite si l'information n'existe pas dans la base.

  • LangChain
